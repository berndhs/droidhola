ksba_reader_read: : Readder object : A buffer for returning the data : The length of this buffer : Number of bytes actually read.

Read data from the current read position to the supplied , max. bytes are read and the actual number of bytes read are returned in . If there are no more bytes available GPG_ERR_EOF is returned and is set to 0.

If a of NULL is specified, the function does only return the number of bytes available and does not move the read pointer. This does only work for objects initialized from memory; if the object is not capable of this it will return the error GPG_ERR_NOT_IMPLEMENTED

ksba_reader_set_cb: : Reader object : Callback function : Value passed to the callback function

Initialize the reader object with a callback function. This callback function is defined as: <literal> typedef int (*cb) (void *cb_value, char *buffer, size_t count, size_t *nread); </literal>

The callback should return a maximium of bytes in and the number actually read in . It may return 0 in if there are no bytes currently available. To indicate EOF the callback should return with an error code of GPG_ERR_EOF and set to 0. The callback may support passing NULL for and and %0 for count as an indication to reset its internal read pointer.

Return value: 0 on success or an error code
